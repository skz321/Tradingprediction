Introduction¶

The Problem¶

I wanted to see if I could use machine learning to predict whether my trades would be profitable or not. This is a classification problem - basically trying to predict if a trade will make money (1) or lose money (0).

What I'm Trying to Figure Out¶

  • Can machine learning actually predict trade profitability?
  • Which factors matter most for successful trades?
  • Are there patterns I can spot in winning vs losing trades?

Why This Matters¶

If I can predict which trades are more likely to be profitable, I could:

  • Avoid bad trades before I make them
  • Focus on high probability setups
  • Reduce my overall losses
  • Make more informed trading decisions

The Data¶

Where It Came From¶

I used my own trading records from August -September. It's real data from actual trades I made, so it's pretty realistic for testing this idea.

What I have:

  • 1,153 total trading records
  • 751 trades that actually got filled (65% fill rate)
  • Mix of buy and sell orders across different stocks

What's in the Data¶

Basic Trade Info:

  • Symbol: Which stock (NVDA, NKE, MARA)
  • Side: Buy or Sell
  • Price: What I paid (like $0.36, $0.76)
  • Status: Whether the trade went through or got cancelled
  • Time: When I placed and filled the trade

The Challenge: I don't have exit prices, so I can't calculate real profitability. I'll need to create a synthetic target based on trade characteristics.

Data Quality¶

  • Some missing prices (354 records) - these were cancelled orders
  • Price range from $0.36 to $8.40
  • 389 different stocks, so pretty diversified
  • Most trading happened during market hours (9-10 AM, 1-3 PM)

Preprocessing¶

What I Had to Fix¶

Missing Data Issues:

  • 354 trades missing average prices (cancelled orders)
  • 402 trades missing fill times (never executed)
  • Solution: Focus only on filled trades (751 records)

Price Data Problems:

  • Prices had "@" symbols (like "@0.360")
  • Some negative prices (probably data entry errors)
  • Solution: Clean the data and convert to numbers

Features I Created¶

Price Features:

  • Cleaned prices (removed @ symbols)
  • Log-transformed prices (handles skewed distributions)
  • Normalized prices (standardized for comparison)

Trade Features:

  • Buy = 1, Sell = 0
  • GTC = 1, DAY = 0
  • Hour of day

Target Variable: Since I don't have real exit prices, I created a synthetic profitability target using trade characteristics and some randomness.

Why These Steps Matter¶

  • Data Quality: Only use complete, reliable data
  • Feature Engineering: Transform raw data into useful predictors
  • Scaling: Make sure all features are on the same scale
  • Target Creation: Need something to predict

The Models I Used¶

Why These Two Algorithms?¶

I picked two different approaches to compare:

Logistic Regression:

  • Simple, interpretable linear model
  • Good baseline to start with
  • Shows which features matter most
  • Fast and easy to understand

Random Forest:

  • Can handle complex, non linear patterns
  • Good at finding feature interactions
  • Provides feature importance rankings
  • More sophisticated but harder to interpret

How They Work¶

Logistic Regression:

  • Uses a linear combination of features
  • Applies sigmoid function to get probabilities
  • Simple decision boundary
  • Easy to interpret coefficients

Random Forest:

  • Builds many decision trees
  • Each tree votes on the final prediction
  • Can capture complex patterns
  • Shows which features are most important

Why This Approach Makes Sense¶

  • Comparison: See which method works better
  • Interpretability: Logistic regression is easy to understand
  • Complexity: Random forest can find hidden patterns
  • Validation: Two different approaches validate results

How Well Did the Models Perform?¶

The Results¶

Logistic Regression:

  • Accuracy: 56.3%
  • Precision: 32.9% (lots of false positives)
  • Recall: 56.3% (catches more profitable trades)
  • F1-Score: 41.5%

Random Forest:

  • Accuracy: 53.0%
  • Precision: 52.1% (much better at avoiding false positives)
  • Recall: 53.0% (misses some profitable trades)
  • F1-Score: 52.4%

What This Means¶

Logistic Regression:

  • More optimistic - predicts profitable for most trades
  • Catches more actual profitable trades
  • But has lots of false positives
  • Good for not missing opportunities

Random Forest:

  • More conservative - only predicts profitable when confident
  • Better at avoiding bad predictions
  • Misses some profitable trades
  • Good for risk management

Feature Importance (Random Forest)¶

  1. Price normalization (most important)
  2. Trade direction (buy vs sell)
  3. Log-transformed price

Key Insight: Price-related features dominate everything else.

Can ML predict profitability? Yes, but with limitations. 53-56% accuracy is better than random (50%) but leaves room for improvement. The Random Forest model is right 52% of the time when it predicts profitable.

Which factors matter most? Price related features dominate:

  2. Trade direction (buy vs sell)
  3. Log-transformed price patterns

Impact¶

Positive Impacts¶

Leveling the Playing Field: Machine learning tools can give individual traders access to sophisticated analysis that was previously only available to large firms. This could help reduce the advantage that institutional traders have.

Risk Management: Predictive models can help traders avoid high-risk trades and reduce financial losses. This protects individual investors from significant harm.

Market Efficiency: Better-informed trading decisions could lead to more efficient price discovery and market functioning.

Potential Problems¶

Bias and Discrimination: Models trained on historical data might perpetuate past biases or discriminate against certain trading strategies. The "black box" nature makes this hard to detect.

Market Manipulation: Sophisticated models could be used to manipulate markets or exploit other traders. There's also the risk of unequal access creating unfair advantages.

Overconfidence: Traders might become overconfident in model predictions, leading to excessive risk-taking or neglecting fundamental analysis.

Wealth Inequality: Advanced models could concentrate wealth among those with access to sophisticated technology, making the gap between retail and institutional traders even wider.

How to Handle These Issues¶

Transparency:

  • Make model decisions explainable
  • Document limitations and uncertainty
  • Share methodology when possible

Human Oversight:

  • Use models to augment, not replace, human judgment
  • Require human approval for high-stakes decisions
  • Maintain human accountability

Fairness:

  • Regularly check for bias in model predictions
  • Ensure diverse training data
  • Provide equal access to predictive tools

The Bottom Line¶

While machine learning in trading offers benefits, it also creates ethical challenges. The key is implementing these technologies responsibly with proper safeguards, transparency, and human oversight.
